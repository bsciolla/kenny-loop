# `<project-manage>` task description

Inspect the "overarching goal" and the progress made.

- Decide what next step should be taken in the "superplan". This may mean spawning whole new phases (design → implement → test → ...) if a stumbling block is reached.
- Make new `<project-manage>`, `<design>`, `<implement>`, `<test>`, `<report>` instruction lines (or edit existing lines).
- Write a new `[slug-description.md]` file for each line, describing the detail of the actions needed for the task to be successful.
- Rely on thinking done during the `<design>`.
- Always add a `<report>` in the end.
- Write the tasks in the superplan following the current conventions.
